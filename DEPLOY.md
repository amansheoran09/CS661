# Deploying CryptoLens

The entire stack is deployed on Render using a single Blueprint. This ensures all services remain closely collocated in the same region and avoids network routing issues that can occur with certain edge network IPs.

| Service | Host | Why |
|---|---|---|
| `frontend` | **Render** | Static Vite SPA, hosted as a static site via the Blueprint. |
| `backend` (FastAPI + DuckDB) | **Render** | Long-running process; data baked in at build time. |
| `ws-server` (live order book) | **Render** | Needs a persistent WebSocket connection to Binance. |

You need the code on GitHub first (`git init`, push).

---

## 1. Deploying the Full Stack on Render

All three services are defined in [`render.yaml`](render.yaml) and are deployed together.

1. Render Dashboard > **New > Blueprint**, then connect this GitHub repo.
2. Render reads `render.yaml` and creates **`cryptolens-api`**, **`cryptolens-ws`**, and **`cryptolens-frontend`**. Click **Apply**.
3. The first build of `cryptolens-api` runs the data ingestion (default: 2020-01 to 2024-12 for all 5 assets) and bakes the DuckDB file into the deploy. To load a shorter history, edit the `INGEST_START` / `INGEST_END` env vars in your Render dashboard or `render.yaml` (note: full 5-year builds take noticeably longer).
4. For the frontend, Render will automatically inject the `VITE_API_BASE` and `VITE_WS_URL` environment variables at build time based on the URLs defined in the blueprint. 
5. When all services are live, you can visit your application via the frontend URL generated by Render.

> Free-tier Render services sleep after 15 min idle; the first API request cold-starts (30–60 s).

Sanity-check the API: open `https://<your-api-url>.onrender.com/api/health`; it should return `{"status":"ok"}`.

## Notes
- To lock CORS down: set `CORS_ORIGINS=https://<your-frontend-app>.onrender.com` on the
  Render `cryptolens-api` service.
- Local dev is unchanged: leave the two `VITE_*` vars unset and run
  `scripts/start-all.ps1`; the app falls back to the Vite proxy and
  `ws://localhost:8080`.
